# Account Plan — Top 3 (dev-productivity SaaS)

Generated from `leads.csv` (pith enrichment, public data only). Selling into engineering-led orgs that are actively hiring devs => budget + the exact pain (dev velocity) we sell.

## 1. Stripe — fit 7 (top)

- **Why:** Highest reachability + relevance in the set. Public GitHub org (`github.com/stripe`), active careers page, LinkedIn + Twitter, and pith surfaced 5 company-matched emails including `sales@stripe.com` and `careers@stripe.com`. Large eng org = many seats.
- **Entry contact:** `sales@stripe.com` for a routed intro; better path is a named eng-leader via LinkedIn (`linkedin.com/company/stripe`). Do NOT use `accommodations@stripe.com` — that is an ADA/hiring accommodation mailbox pith scraped from the careers flow, not a sales channel.
- **Angle:** They ship developer tooling themselves, so they feel dev-velocity pain acutely and respect tools that measure it. Lead with a metric (PR cycle time / CI wait) benchmarked against other infra companies.

## 2. Linear — fit 6

- **Why:** Engineering-led (public GitHub `github.com/linear`), careers page live, and pith found the cleanest entry emails of the whole set: `sales@linear.app` and `hello@linear.app`. Small enough that a single email actually reaches a decision-maker.
- **Entry contact:** `hello@linear.app` (founder-reachable at their size) or `sales@linear.app`.
- **Angle:** Linear sells a "speed for eng teams" product — mirror their own thesis back: "you sell velocity to eng teams; here's what your own team's velocity looks like." Peer-to-peer, not vendor-to-buyer.

## 3. Vercel — fit 5

- **Why:** Public GitHub (`github.com/vercel`), careers page, full social footprint. No email surfaced by pith (gap — see README), so this is a social-first play.
- **Entry contact:** No direct email found. Route via LinkedIn (`linkedin.com/company/vercel`) or an eng-manager on their GitHub org.
- **Angle:** Next.js shop (pith detected the framework) with a large, fast-growing eng team. Pitch: keep DX high as headcount scales — the phase where velocity tooling pays back fastest.
